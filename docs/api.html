API

Methods

Renderers

This is the list of methods we can call on renderers.
Check out the documentation about renderers to get more informations.

Methods Description
Renderer.onEnter() This methods is called when the data-router-view is added to the DOM.
Renderer.onLeave() This methods is called when transition to hide the data-router-view is called.
Renderer.onEnterCompleted() This methods is called when the transition to show the data-router-view is over.
Renderer.onLeaveCompleted() This methods is called when the data-router-view is removed from the DOM.

Transitions

This is the list of methods we can call on transitions.
Check out the documentation about transitions to get more informations.

Methods Description
Transition.in(view, done) This method is called to show the data-router-view.
Transition.out(view, done) This method is called to hide the data-router-view.

Core

Highway is based on the tiny-emitter library so when we call Highway.Core we have access to all the methods available in tiny-emitter in order to use the events listed below. In addition to the methods inherited from tiny-emitter, we have some buil-in ones available we might need:

Methods Description
Core.attach() This method listen to all links of the document that have no target or data-router-disabled attribute and calls Highway on click.
Core.detach() This method clears the events attached to all the links of the document.
Core.redirect(href) This method navigates to a given URL using Highway.

Events

Events Description
NAVIGATE_IN This event is emitted everytime the transition to show the data-router-view is called. It returns those arguments:

  • to: The data-router-view we show.
  • location: The URL datas of the current view.
NAVIGATE_OUT This event is emitted everytime the transition to hide the data-router-view is called. It returns those arguments:

  • from: The data-router-view we hide.
  • location: The URL datas of the current view.
NAVIGATE_END This event is emitted everytime a transition ends. It returns those arguments:

  • to: The data-router-view we show.
  • from: The data-router-view we hide.
  • location: The URL datas of the current view.
